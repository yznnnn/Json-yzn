

# 塔吊算法 - 容量变换策略

设定参数 $r = 0.25$

假设一个`myString`对象当前存储的字符串需要 $n$ 块内存，那么将为其分配 $n(1+r)$ 块内存的空间，记作 ${\rm max}$。

定义监测范围为：$[\frac{1-r}{1+r}{\rm max},{\rm max}]$

若进行某些操作之后，该对象所存储字符串需要的内存变为 $m$ 块：

* $m\in(\frac{1-r}{1+r}{\rm max},{\rm max})$

  不需要更新内存容量

* $m\not\in(\frac{1-r}{1+r}{\rm max},{\rm max})$

  更新内存空间大小为  $m(1+r)$  块
